# 交互设计改进实施文档

本文档整理本项目交互体验的 7 项改进方向，并作为本轮实现的验收依据。

## 设计目标

保留项目最强的“后末日科幻 HUD 档案系统”气质，同时减少重复访问和高频浏览时的摩擦。桌面端继续强调沉浸、仪式感和系统反馈；移动端转向更高效率的浏览、导航和阅读。

## 1. 首屏仪式感与二次访问快进

目标：第一次访问保留完整启动仪式，二次访问不再强制等待完整终端动画。

实现要求：

- 使用 `sessionStorage` 记录本会话已经完成过启动动画。
- 二次访问自动进入 fast boot 模式，缩短加载时长和后续列动画时长。
- 在加载屏提供明确但不喧宾夺主的 `SKIP BOOT` 控件。
- 快进后仍然触发 `handleLoadingComplete`，保持主界面状态一致。

验收标准：

- 首次访问仍有完整启动动画。
- 同一浏览器会话中刷新或返回时明显更快进入主界面。
- 键盘用户可以聚焦并触发跳过按钮。

## 2. 导航可供性增强

目标：六列 HUD 导航仍保持独特，但新用户能更快判断“这里可点击、每列是什么”。

实现要求：

- 为首页六列补充 hover/focus 状态下的横向信息标签。
- 列元素支持键盘聚焦、Enter/Space 激活。
- 不改变现有 `/content#works|experience|blog|life|contact|about` 契约。
- 移动端保留清晰的栏目文字，不依赖 hover。

验收标准：

- 鼠标 hover 和键盘 focus 都能看到栏目说明。
- 屏幕阅读器能读出栏目名称和用途。
- 加载动画未完成前仍不可误触。

## 3. Motion hierarchy

目标：保留进入系统的重仪式感，降低内容内高频跳转的等待成本。

实现要求：

- 首屏启动和首次列展开保持偏完整。
- 内容页与详情页之间的转场缩短。
- 详情页关闭、同级详情切换、返回主内容时使用更轻的节奏。
- 遵守 `prefers-reduced-motion`。

验收标准：

- 高频看多个项目时不再明显拖沓。
- 快进启动后列动画也随之缩短。
- 减少动画模式下不会出现长时间等待。

## 4. 电量、拉杆和反色模式反馈

目标：让用户知道自己触发了什么，而不是只看到一个隐藏彩蛋。

实现要求：

- 增加全局系统反馈组件，用短促 HUD 文案反馈关键状态。
- 拉杆触发充电、尝试放电、满电反色、放电恢复时都有状态提示。
- 不用长说明，不破坏沉浸感。
- 反馈文案可被屏幕阅读器感知。

验收标准：

- 触发拉杆后 1-2 秒内能看到状态反馈。
- 满电进入反色模式时有明确系统提示。
- 非满电尝试放电时能提示需要 100% 能量。

## 5. 详情页档案叙事模板

目标：让 Web/Life/Blog 详情页更像统一的档案系统，而不是三套相似但松散的页面。

实现要求：

- Web 详情统一为 `SIGNAL / META / BRIEF / ARCHIVE / OUTPUT`。
- Life 详情统一为 `SIGNAL / META / BRIEF / ARCHIVE / OUTPUT`，Meta 使用现有标签、图库和链接数量生成。
- Blog 详情统一为 `SIGNAL / META / BRIEF / OUTPUT`，文章日期、阅读时间、标签和摘要归入 Meta。
- 右侧锚点导航使用 `Signal / Meta / Brief / Archive / Output` 短标签，页面区块标题统一为系统式命名。

验收标准：

- 三类详情页结构更一致。
- 不重写详情页架构，只在现有结构上统一标签、状态和导航。

## 6. 移动端效率优先

目标：移动端不照搬桌面端的复杂仪式感，而是优先让用户快速浏览和返回。

实现要求：

- 底部功能栏补充可见/可读的语义标签。
- 抽屉导航和底部栏保持明确的当前状态。
- 移动端减少非必要装饰遮挡和过密 HUD 信息。
- 卡片、详情页、Lightbox 保持手指操作友好。

验收标准：

- 主要导航不依赖 hover。
- 底部栏按钮有明确 aria-label 和可见短标签。
- 移动端内容区域阅读负担下降。

## 7. 无障碍与键盘路径

目标：让自定义光标、空按钮、hover-only 效果不阻断键盘和辅助技术用户。

实现要求：

- 所有可点击元素具备语义、`aria-label` 或可见文本。
- 空按钮用 `aria-label` 描述功能。
- 关键交互支持 Enter/Space。
- 增加全局 focus-visible 样式。
- 对 hover-only 的信息也提供 focus 等价路径。

验收标准：

- Tab 可以走通首页导航、左侧导航、底部栏、卡片、详情页返回、Lightbox。
- focus 状态可见。
- 不再出现无法理解用途的空按钮。

## 本轮实现边界

- 保留项目现有视觉方向和技术架构。
- 不引入新的 UI 组件库。
- 不迁移 App Router。
- 不删除 SSE 统计、自定义服务器、WebGL 背景、电量系统或音乐播放器。
- 构建前需要通过项目本地类型检查；`mdx/types` 依赖已改为本地轻量类型，避免包管理器差异导致构建失败。
