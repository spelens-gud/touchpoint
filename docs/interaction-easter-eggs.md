# 交互小巧思与彩蛋实施文档

本文档整理本轮 10 个轻量交互彩蛋，并作为实现与验收依据。所有彩蛋都应服务于项目既有的后末日科幻 HUD 气质：短促、克制、可感知，但不阻断主路径。

## 设计原则

- 彩蛋只能增强反馈，不隐藏核心功能。
- 鼠标、键盘和触屏用户都应有等价或不受影响的路径。
- 文案保持终端式、英文大写、短句表达。
- 动效遵守 `prefers-reduced-motion`，减少动画用户获得安静模式回应。
- 普通模式与 `.inverted` 模式都要可读，不新增并行设计系统。

## 1. 启动屏隐藏指令

在启动终端或首页加载阶段监听用户输入 `whoami`、`status`、`rain`、`help`，把结果作为终端日志追加。它像一次系统握手，不需要输入框，也不改变加载流程。

验收标准：输入完整命令后终端出现对应回应；`status` 反映当前启动进度；跳过启动和二次 fast boot 不受影响。

## 2. 电量阈值微反馈

电量低于 30% 时通过全局 HUD 提示 `LOW RESERVE`；恢复到稳定区间提示 `POWER STABLE`；100% 继续沿用反色/满电系统反馈。播放器把手和唱臂同步体现低电或满电状态。

验收标准：电量状态变化时出现短反馈；低电、稳定、满电视觉状态可区分但不刺眼。

## 3. 首页六列二次悬停彩蛋

同一导航列在短时间内第二次 hover 或 focus 时显示一句隐藏短句，例如 `THINGS THAT SURVIVED`。这是对探索型用户的轻奖励。

验收标准：首次 hover/focus 仍显示原说明；二次触发才出现彩蛋；键盘 focus 可触发；移动端不依赖 hover。

## 4. 访客数联动 HUD

使用现有实时统计，在访客数变化时显示 `SOLO SIGNAL`、`PARALLEL PRESENCE DETECTED` 或 `NEW SIGNAL JOINED`。它让在线人数从数字变成环境事件。

验收标准：首次读取和人数增加时有全局 HUD 反馈；不重复刷屏；不影响 SSE 统计能力。

## 5. Blog 阅读进度信号强度

博客详情页右侧导航增加阅读进度信号条，阅读到末尾显示 `TRANSMISSION COMPLETE`。它替代常规进度条的直白表达。

验收标准：滚动文章时信号条更新；接近末尾文案切换；移动端不遮挡正文。

## 6. Contact 雷达隐藏回声

复制邮箱后不只显示 `Copied`，而是显示随机 HUD 回执；连续复制 3 次及以后显示 `YOU ALREADY HAVE THE COORDINATES`。

验收标准：复制成功后反馈可见并可被辅助技术感知；多次复制有递进文案；失败时仍有清晰提示。

## 7. Lightbox 档案扫描感

图片灯箱显示当前图像序号和 `ARCHIVE VERIFIED` 档案印章，让图库更像档案扫描器。

验收标准：打开任意图库图片都能看到序号与验证状态；不遮挡主要图片和关闭按钮。

## 8. Logo/avatar 校准失败彩蛋

左侧头像区域在 hover 或 keyboard focus 时显示 `IDENTITY MODULE EMPTY`，像一次身份模块校准失败。该区域不作为导航入口，只提供非阻塞反馈。

验收标准：头像区域可聚焦且有语义标签；提示只在 hover/focus 时显示；不影响左侧面板其他控件。

## 9. Reduced-motion 专属回应

当用户系统开启减少动画时，启动终端追加 `MOTION REDUCED // QUIET MODE ENABLED`。这是对用户偏好的可见尊重。

验收标准：减少动画模式不触发长动画；终端日志出现安静模式回应。

## 10. 午夜模式

本地时间 00:00-04:00 自动进入 `NIGHT WATCH ACTIVE` 氛围状态，降低辉光、略收暗网格，并用一次 HUD 反馈提示夜巡模式。

验收标准：午夜时段容器增加夜间状态；白天不显示；不改变导航与内容结构。

## 实现边界

- 不新增 UI 库或包管理器。
- 不迁移 App Router。
- 不删除 `server.js`、SSE 统计、自定义光标、WebGL 背景或电量系统。
- 所有新增文案保持短句和 HUD 风格。
